# 高德地图 Web 服务 API 接口文档

本文档整理自高德开放平台 Web 服务 API 规范，用于本项目实现相遇地点推荐（地理编码、周边 POI 搜索等）。高德 MCP Server（如 `@amap/amap-maps-mcp-server`）底层也基于这些 Web 服务接口，后端可通过 HTTP 直接调用实现相同能力。

---

## 一、通用规范

### 1.1 鉴权

- 使用 **Web 服务 API 类型 Key** 鉴权
- 申请入口：[高德开放平台 - 控制台](https://console.amap.com/dev/key/app)
- 请求时以 `key` 参数传递

### 1.2 请求方式

- 所有接口均为 **GET**
- 参数通过 URL 查询字符串传递，UTF-8 编码

### 1.3 返回格式

- 默认 JSON，可通过 `output` 参数指定 XML

---

## 二、地理编码 API

将结构化地址或地标名称转换为经纬度坐标。

### 2.1 基本信息

- **URL**：`https://restapi.amap.com/v3/geocode/geo`
- **方法**：GET

### 2.2 请求参数

| 参数    | 必填 | 说明                                           |
|---------|------|------------------------------------------------|
| key     | 是   | 高德 Key                                       |
| address | 是   | 地址，如：北京市朝阳区阜通东大街6号；或地标如：西单、国贸 |
| city    | 否   | 限定城市，可选：城市中文、citycode、adcode     |

### 2.3 响应示例

```json
{
  "status": "1",
  "count": "1",
  "info": "OK",
  "geocodes": [
    {
      "location": "116.375252,39.907883",
      "formatted_address": "北京市西城区西单",
      "province": "北京市",
      "city": "北京市",
      "district": "西城区",
      "adcode": "110102",
      "level": "热点商圈"
    }
  ]
}
```

- `location`：经度,纬度
- `status` 为 `1` 表示成功，`0` 表示失败

## 三、逆地理编码 API

将经纬度转换为结构化地址及周边信息。

### 3.1 基本信息

- **URL**：`https://restapi.amap.com/v3/geocode/regeo`
- **方法**：GET

### 3.2 请求参数

| 参数      | 必填 | 说明                                           |
|-----------|------|------------------------------------------------|
| key       | 是   | 高德 Key                                       |
| location  | 是   | 经纬度，格式：经度,纬度                        |
| radius    | 否   | 搜索半径（米），0–3000，默认 1000              |
| extensions| 否   | `base`（默认）或 `all`，all 时返回周边 POI 等  |

---

## 四、周边搜索 API（POI 周边）

在指定经纬度附近搜索 POI（餐厅、咖啡、商场等）。

### 4.1 基本信息

- **URL**：`https://restapi.amap.com/v3/place/around`
- **方法**：GET

### 4.2 请求参数

| 参数     | 必填 | 说明                                           |
|----------|------|------------------------------------------------|
| key      | 是   | 高德 Key                                       |
| location | 是   | 中心点经纬度，格式：经度,纬度                  |
| keywords | 否   | 关键词，如：咖啡、餐厅、商场（与 types 二选一）|
| types    | 否   | POI 类型码，如：050000（餐饮）（与 keywords 二选一）|
| radius   | 否   | 搜索半径（米），默认 3000，最大 50000          |
| offset   | 否   | 每页条数，建议不超过 25，默认 20               |
| page     | 否   | 页码，默认 1                                   |

### 4.3 响应示例

```json
{
  "status": "1",
  "count": "20",
  "info": "OK",
  "pois": [
    {
      "id": "B000A8UIN8",
      "name": "星巴克",
      "type": "咖啡厅",
      "location": "116.37,39.90",
      "address": "西单北大街131号",
      "distance": "500",
      "tel": "010-xxx"
    }
  ]
}
```

- `location`：POI 经纬度
- `distance`：到中心点的距离（米）

## 五、路径规划 API（步行/驾车）

计算两点间路线、距离、耗时，可用于验证见面地点是否合适。

### 5.1 步行路径规划

- **URL**：`https://restapi.amap.com/v3/direction/walking`
- **方法**：GET

| 参数        | 必填 | 说明                    |
|-------------|------|-------------------------|
| key         | 是   | 高德 Key                |
| origin      | 是   | 起点经纬度，经度,纬度   |
| destination | 是   | 终点经纬度，经度,纬度   |

### 5.2 驾车路径规划

- **URL**：`https://restapi.amap.com/v3/direction/driving`
- **方法**：GET

参数与步行类似，额外支持 `waypoints`（途经点）、`strategy`（驾车策略）等。

---

## 六、相遇地点推荐流程（本项目）

1. **地理编码**：将「我的位置」「朋友的位置」转为经纬度
2. **计算中点**：`mid_lon = (lon1 + lon2) / 2`，`mid_lat = (lat1 + lat2) / 2`
3. **周边搜索**：以中点为圆心，搜索咖啡、餐厅等 POI 作为推荐见面地点
4. （可选）**路径规划**：计算双方到推荐地点的距离与时间
