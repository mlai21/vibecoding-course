# 阿里云百炼平台接口文档

本文档整理自阿里云百炼（DashScope）官方开发规范，用于本项目对接语音识别（ASR）等能力。

---

## 一、通用规范

### 1.1 鉴权方式

- 使用 **API Key** 鉴权，在请求头中添加：`Authorization: Bearer {api-key}`
- API Key 获取： [阿里云百炼 - 获取 API Key](https://www.alibabacloud.com/help/zh/model-studio/get-api-key)

### 1.2 地域与接入点

| 地域     | 接入点                     | 说明                         |
|----------|----------------------------|------------------------------|
| 中国内地 | `dashscope.aliyuncs.com`   | 数据与计算均在中国内地       |
| 国际     | `dashscope-intl.aliyuncs.com` | 数据存储在新加坡，计算全球调度 |

### 1.3 调用方式限制

- **不支持前端直连**：需通过后端中转调用
- **HTTP 请求**：使用标准 HTTP/HTTPS，不强制使用 SDK

## 二、Fun-ASR 录音文件识别（ASR）

### 2.1 概述

- **服务**：录音文件语音转文字
- **模型**：`fun-asr`（当前等同 fun-asr-2025-11-07）
- **调用流程**：先提交任务 → 轮询查询任务状态 → 任务完成后从 `transcription_url` 下载结果

### 2.2 输入约束

- **不支持**：本地文件直传、Base64 编码
- **支持**：公网可访问的 **HTTP/HTTPS URL** 或 **OSS 临时 URL**（`oss://` 前缀）
- 使用 `oss://` 时，请求头必须加：`X-DashScope-OssResourceResolve: enable`

### 2.3 支持格式与限制

| 项目       | 说明                                                                 |
|------------|----------------------------------------------------------------------|
| 音频格式   | aac, amr, avi, flac, flv, m4a, mkv, mov, mp3, mp4, mpeg, ogg, opus, wav, webm, wma, wmv |
| 采样率     | 任意                                                                 |
| 文件大小   | 不超过 2GB                                                           |
| 时长       | 12 小时以内                                                          |
| 单次请求   | 最多 100 个文件 URL                                                  |

### 2.4 接口一：提交转写任务

**URL**

- 中国内地：`https://dashscope.aliyuncs.com/api/v1/services/audio/asr/transcription`
- 国际：`https://dashscope-intl.aliyuncs.com/api/v1/services/audio/asr/transcription`

**方法**：`POST`

**请求头**

```http
Authorization: Bearer {api-key}
Content-Type: application/json
X-DashScope-Async: enable
X-DashScope-OssResourceResolve: enable   # 使用 oss:// URL 时必填
```

**请求体**

```json
{
  "model": "fun-asr",
  "input": {
    "file_urls": [
      "https://example.com/audio.wav",
      "oss://bucket/path/audio.webm"
    ]
  },
  "parameters": {
    "channel_id": [0],
    "language_hints": ["zh"],
    "diarization_enabled": false
  }
}
```

| 参数               | 类型   | 必填 | 说明                                      |
|--------------------|--------|------|-------------------------------------------|
| model              | string | 是   | 模型名，如 `fun-asr`                      |
| input.file_urls    | array  | 是   | 待识别文件 URL 列表                       |
| parameters.channel_id | array | 否   | 音轨索引，默认 [0]                        |
| parameters.language_hints | array | 否 | 语种，如 ["zh"]（中文）、["en"]（英文）   |
| parameters.diarization_enabled | bool | 否 | 说话人分离，默认 false                    |

**响应**

```json
{
  "output": {
    "task_status": "PENDING",
    "task_id": "xxx-xxx-xxx"
  },
  "request_id": "xxx"
}
```

### 2.5 接口二：查询任务结果

**URL**

- 中国内地：`https://dashscope.aliyuncs.com/api/v1/tasks/{task_id}`
- 国际：`https://dashscope-intl.aliyuncs.com/api/v1/tasks/{task_id}`

**方法**：`POST`

**请求头**

```http
Authorization: Bearer {api-key}
```

**响应（成功）**

```json
{
  "output": {
    "task_id": "xxx",
    "task_status": "SUCCEEDED",
    "results": [
      {
        "file_url": "https://...",
        "transcription_url": "https://dashscope-result-bj.oss-cn-beijing.aliyuncs.com/...",
        "subtask_status": "SUCCEEDED"
      }
    ]
  }
}
```

- `transcription_url`：识别结果 JSON 的下载地址，有效期 24 小时

**识别结果 JSON 结构**

```json
{
  "file_url": "https://...",
  "properties": {
    "audio_format": "pcm_s16le",
    "original_sampling_rate": 16000,
    "original_duration_in_milliseconds": 3834
  },
  "transcripts": [
    {
      "channel_id": 0,
      "text": "转写文本内容",
      "sentences": [
        {
          "begin_time": 100,
          "end_time": 3820,
          "text": "句子级文本",
          "words": [...]
        }
      ]
    }
  ]
}
```

## 三、文件上传至临时存储（获取 oss:// URL）

ASR 需要文件 URL，本地文件需先上传到百炼临时 OSS，获取 `oss://` 链接。

### 3.1 步骤一：获取上传凭证

**URL**：`https://dashscope.aliyuncs.com/api/v1/uploads`

**方法**：`GET`

**请求头**

```http
Authorization: Bearer {api-key}
Content-Type: application/json
```

**查询参数**

| 参数  | 说明                          |
|-------|-------------------------------|
| action | 固定为 `getPolicy`            |
| model | 目标模型，如 `fun-asr`（需与后续 ASR 调用一致） |

**响应**

```json
{
  "data": {
    "upload_host": "https://...",
    "upload_dir": "dashscope-instant/xxx/2024-xx-xx/xxx",
    "oss_access_key_id": "...",
    "signature": "...",
    "policy": "...",
    "x_oss_object_acl": "...",
    "x_oss_forbid_overwrite": "..."
  }
}
```

### 3.2 步骤二：上传文件到 OSS

**URL**：`upload_host`（来自上一步）

**方法**：`POST`

**Content-Type**：`multipart/form-data`

**表单字段**

| 字段                   | 类型 | 说明              |
|------------------------|------|-------------------|
| OSSAccessKeyId         | 文本 | 来自凭证          |
| Signature              | 文本 | 来自凭证          |
| policy                 | 文本 | 来自凭证          |
| x-oss-object-acl       | 文本 | 来自凭证          |
| x-oss-forbid-overwrite | 文本 | 来自凭证          |
| key                    | 文本 | `{upload_dir}/{文件名}` |
| success_action_status  | 文本 | 固定 `200`        |
| file                   | 文件 | 音频文件          |

**临时 URL**：`oss://{upload_dir}/{文件名}`

### 3.3 临时存储限制

- 有效期：48 小时
- 文件与模型绑定，上传时指定的 model 须与 ASR 调用一致
- 限流：100 QPS，不适合生产高并发
